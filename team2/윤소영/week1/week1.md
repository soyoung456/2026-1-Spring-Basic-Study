## 프로젝트

- 하나의 목표를 세우고, 달성하기 위한 모든 과정
- 일정 기간, 일정 목적을 달성하기 위한 업무 묶음 (프로세스)
- 하나의 프로그램, 작은 기능, 에러 해결, 기능 추가 등

## 왜 스프링인가

- 스프링을 사용하면 자바로 쉽게 개발 가능
- 스프링은 자바 **프레임워크**

- framework
    - 틀 안에서 일 하는 것
    - 틀 안에 필요한 기능 미리 구현 → 개발자 편리
    - 다양한 라이브러리까지 편리하게 사용 가능
    - 개발자는 로직에 집중해서 개발 가능
- 라이브러리 = library
    - 선배 개발자가 만들어 둔 코드 빌리기

## 스프링 프레임워크

- 자바 개발자로서 불편했던 점, 프로젝트 시 필요했던 기능 등을 해결하기 위함
- 스프링 프레임워크가 많은 것을 제공해 주지만…
    - 설정할 것이 너무 많음
    - 웹, 앱에 특화해서 만들고 싶은 니즈

## Spring Boot

- 스프링 프레임워크의 웹 특별판
- **웹에 특화**된 스프링 but 웹만 할 수 있는 것은 아님
- 많은 설정들이 자동화
